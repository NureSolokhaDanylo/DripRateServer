### Sprint Backlog: DripRate Server (Detailed)

| Категория (Приоритет) | Заметки (Бизнес-смысл) | Детали UI и API (Элементы, Переходы, Взаимодействия) |
| :--- | :--- | :--- |
| **1. Аутентификация**<br>Регистрация | Создание нового аккаунта. | **UI:** Форма (email, логин, пароль). Кнопка "Зарегистрироваться".<br>**API:** `POST /api/Auth/register`.<br>**Переход:** Редирект на Главный экран (Лента) + сохранение токена локально. |
| **2. Аутентификация**<br>Вход | Доступ к существующему аккаунту. | **UI:** Форма (логин/email, пароль). Кнопка "Войти".<br>**API:** `POST /api/Auth/login`.<br>**Переход:** Редирект на Главный экран (Лента) + сохранение токена. |
| **3. Аутентификация**<br>Смена пароля | Безопасность профиля. | **UI:** Форма в настройках (старый пароль, новый пароль).<br>**API:** `PUT /api/Auth/password`.<br>**Взаимодействие:** Toast/уведомление об успешной смене. |
| **4. Лента (Feed)**<br>Глобальная лента | Основной источник контента. | **UI:** Бесконечный список/сетка постов. Карточка: фото, аватар, лайки.<br>**API:** `GET /api/Feed/global` (параметры: `skip`, `take`).<br>**Взаимодействие:** Пагинация при скролле вниз (подгрузка новых элементов). |
| **5. Лента (Feed)**<br>Лента подписок | Контент только от отслеживаемых. | **UI:** Отдельный Tab/Экран Ленты.<br>**API:** `GET /api/Feed/subscriptions`.<br>**Взаимодействие:** Пагинация при скролле. |
| **6. Лента (Feed)**<br>Топ лента | Популярный контент. | **UI:** Отдельный Tab в Ленте.<br>**API:** `GET /api/Feed/top`.<br>**Взаимодействие:** Пагинация при скролле. |
| **7. Публикация**<br>Просмотр деталей | Изучение конкретного поста. | **UI:** Экран/Окно: полноэкранное фото, описание, инфо об авторе, счетчики оценок/сохранений.<br>**API:** `GET /api/Publications/{id}`.<br>**Переход:** Открывается при клике на пост из Ленты или Профиля. |
| **8. Публикация**<br>Поставить/Убрать лайк | Выражение симпатии посту. | **UI:** Иконка сердца на карточке ленты и внутри поста.<br>**API:** `POST /api/Publications/{id}/like` (Toggle-механика).<br>**Взаимодействие:** Оптимистичное обновление UI (сердце красится сразу, число +1/-1). |
| **9. Комментарии**<br>Загрузка и просмотр | Чтение обсуждения под постом. | **UI:** Шторка (BottomSheet) или раздел под фото. Список текстовых блоков с аватарами.<br>**API:** `GET /api/Publications/{id}/comments`.<br>**Переход:** Открытие при клике на иконку комментария в Ленте или Публикации. |
| **10. Комментарии**<br>Написание комментария (Root) | Оставление своего мнения. | **UI:** Инпут ввода текста внизу экрана/шторки комментов.<br>**API:** `POST /api/Publications/{id}/comments` (в DTO `parentCommentId` = null).<br>**Взаимодействие:** Коммент появляется в списке сразу. |
| **11. Комментарии**<br>Ответ на комментарий (Reply) | Древовидное обсуждение. | **UI:** Кнопка "Ответить" под конкретным комментом -> фокус в поле ввода + бейдж "Ответ для @user".<br>**API:** `POST /api/Publications/{id}/comments` (с указанием `parentCommentId`). |
| **12. Комментарии**<br>Удаление комментария | Удаление своего текста. | **UI:** Свайп влево по комменту или меню (три точки) -> Удалить.<br>**API:** `DELETE /api/Publications/{id}/comments/{commentId}`. |
| **13. Комментарии**<br>Лайк на комментарий | Оценка мнения других. | **UI:** Маленькое сердечко возле коммента.<br>**API:** `POST /api/Publications/{id}/comments/{commentId}/like` (Toggle).<br>**Взаимодействие:** Оптимистичное изменение цвета и цифры лайков. |
| **14. Оценки (Assessments)**<br>Просмотр оценок поста | Кто как оценил лук. | **UI:** Шкала/звезды/балл + кнопка посмотреть кто как проголосовал (Список пользователей).<br>**API:** `GET /api/Publications/{id}/assessments` или `GET /api/Publications/{id}/assessments/list`. |
| **15. Оценки (Assessments)**<br>Поставить оценку | Оценить чужой лук. | **UI:** Интерактивный ползунок/кнопки рейтинга в деталях публикации.<br>**API:** `POST /api/Publications/{id}/assessments`.<br>**Взаимодействие:** Обновление средней оценки на экране. |
| **16. Профиль (Мой)**<br>Просмотр профиля | Своя статистика и посты. | **UI:** Шапка (Аватар, ник, фолловеры), Сетка своих публикаций.<br>**API:** Данные: `GET /api/Users/@me`. Публикации: `GET /api/Feed/user/{my_id}`. |
| **17. Профиль (Мой)**<br>Редактирование профиля | Изменение данных. | **UI:** Экран "Настройки профиля". Поля: Имя, Био и т.д.<br>**API:** `PATCH /api/Users/@me`. |
| **18. Профиль (Мой)**<br>Управление аватаром | Обновление/Удаление фото. | **UI:** Клик по аватару в профиле -> "Загрузить новое" / "Удалить".<br>**API:** `PATCH /api/Users/@me/avatar` или `DELETE /api/Users/@me/avatar`. |
| **19. Профиль (Мой)**<br>Настройки предпочтений | Локальные конфиги юзера. | **UI:** Меню Настройки -> Предпочтения.<br>**API:** `GET /api/Users/@me/preferences` и `PUT /api/Users/@me/preferences`. |
| **20. Профиль (Чужой)**<br>Переход и просмотр | Переход на страницу другого пользователя. | **UI:** Аналогично своему, но с кнопкой Follow.<br>**API:** Данные: `GET /api/Users/{id}`. Публикации: `GET /api/Feed/user/{id}`.<br>**Переход:** Откуда угодно: клик по аватару в ленте, в комментах, в списке оценивших, в поиске. |
| **21. Социальное**<br>Подписка / Отписка (Follow) | Следить за юзером. | **UI:** Кнопка "Подписаться" в чужом профиле.<br>**API:** `POST /api/Users/{id}/follow` (Toggle или `DELETE` для отписки). |
| **22. Социальное**<br>Список подписчиков (Followers) | Кто читает профиль. | **UI:** Клик по счетчику "Подписчики" -> Модалка/Экран со списком юзеров.<br>**API:** `GET /api/Users/{id}/followers`.<br>**Взаимодействие:** Можно кликнуть на юзера из списка и перейти в его профиль. |
| **23. Социальное**<br>Список подписок (Following) | Кого читает профиль. | **UI:** Клик по счетчику "Подписки" -> Список.<br>**API:** `GET /api/Users/{id}/following`. |
| **24. Коллекции (Сохранения)**<br>Список коллекций юзера | Просмотр своих папок. | **UI:** Отдельная вкладка "Сохраненки" в Профиле или модалка при сохранении.<br>**API:** `GET /api/Collections/@me` (или `/user/{id}` для чужих, если они публичные). |
| **25. Коллекции (Сохранения)**<br>Создание новой коллекции | Новая папка. | **UI:** Кнопка "+" в списке коллекций. Инпут названия.<br>**API:** `POST /api/Collections`. |
| **26. Коллекции (Сохранения)**<br>Добавление поста в коллекцию | Сохранить лук (Save). | **UI:** Иконка закладки 🔖 на посте. Клик -> Поп-ап выбора коллекции.<br>**API:** `POST /api/Collections/{id}/items/{pubId}`. |
| **27. Коллекции (Сохранения)**<br>Удаление поста из коллекции | Убрать из сохраненного. | **UI:** Снятие выделения с коллекции в поп-апе, либо кнопка "Удалить" внутри коллекции.<br>**API:** `DELETE /api/Collections/{id}/items/{pubId}`. |
| **28. Коллекции (Сохранения)**<br>Просмотр коллекции | Что внутри папки. | **UI:** Экран сетки публикаций.<br>**API:** `GET /api/Collections/{id}`.<br>**Переход:** Клик на публикацию внутри открывает детали публикации. |
| **29. Коллекции (Сохранения)**<br>Удаление коллекции | Снести папку целиком. | **UI:** В меню коллекции (три точки) -> "Удалить".<br>**API:** `DELETE /api/Collections/{id}`. |
| **30. Гардероб (Wardrobe)**<br>Список своих вещей | База для создания луков. | **UI:** Экран/Сетка "Мой гардероб" (Доступно из меню профиля).<br>**API:** `GET /api/Wardrobe`. |
| **31. Гардероб (Wardrobe)**<br>Добавление вещи | Пополнение базы. | **UI:** Форма с загрузкой фото вещи и текстовыми полями (бренд, тип).<br>**API:** `POST /api/Wardrobe`. |
| **32. Гардероб (Wardrobe)**<br>Редактирование/Удаление вещи | Актуализация базы. | **UI:** Экран/шторка деталей вещи -> Кнопки Ред/Удал.<br>**API:** `PUT /api/Wardrobe/{id}` или `DELETE /api/Wardrobe/{id}`. |
| **33. Создание Публикации**<br>Сборка лука (Привязка вещей) | Связывание фото лука и вещей из Гардероба. | **UI:** В процессе создания поста (до отправки на сервер) -> Открытие UI Гардероба (`GET /api/Wardrobe`) -> Выбор вещей (чекбоксы) -> Формирование массива ID вещей для отправки. |
| **34. Создание Публикации**<br>Публикация | Завершение процесса. | **UI:** Кнопка "Опубликовать". Экран загрузки (лоадер).<br>**API:** `POST /api/Publications` (отправка фото, тегов, ID выбранных вещей).<br>**Переход:** По завершению -> Переход на созданный пост `GET /api/Publications/{id}`. |
| **35. Создание Публикации**<br>Удаление своей публикации | Очистка профиля. | **UI:** Кнопка/Меню "Удалить" на своей публикации.<br>**API:** `DELETE /api/Publications/{id}`. |
| **36. Публикация**<br>Просмотр привязанных вещей | Изучение лука из чего он состоит. | **UI:** Блок с тегами/вещами внутри публикации. Клик по вещи -> Шторка с деталями вещи (бренд, фото).<br>**API:** Данные приходят в DTO внутри `GET /api/Publications/{id}` (если нужно больше - реализовать дополнительный запрос, но пока DTO должно хватать). |
| **37. Поиск**<br>Поиск Публикаций | Искать контент. | **UI:** Экран поиска, поле ввода.<br>**API:** `GET /api/Search/publications`.<br>**Взаимодействие:** Дебаунс при вводе. |
| **38. Поиск**<br>Поиск Пользователей | Искать людей. | **UI:** Вкладка "Люди" на экране поиска.<br>**API:** `GET /api/Search/users` (или `GET /api/Users/search`). |
| **39. Модерация**<br>Отправка жалобы (Report) | Репорт контента/юзера. | **UI:** Меню на комменте/посте/профиле -> Пожаловаться -> Выбор причины из списка -> Отправить.<br>**API:** `POST /api/Reports`. |
| **40. Игры (Развлечение)**<br>First Impression | Интерактив 1. | **UI:** Отдельный экран Игр.<br>**API:** Получить: `GET /api/Games/first-impression`. Ответить: `POST /api/Games/first-impression`. |
| **41. Игры (Развлечение)**<br>Guess Price | Интерактив 2. | **UI:** Отдельный экран Игр.<br>**API:** Получить: `GET /api/Games/guess-price`. Ответить: `POST /api/Games/guess-price`. |
| **42. Игры (Развлечение)**<br>Tag Match | Интерактив 3. | **UI:** Отдельный экран Игр.<br>**API:** Получить: `GET /api/Games/tag-match`. Ответить: `POST /api/Games/tag-match`. |
| **43. Реклама**<br>Учет показов в ленте | Бизнес метрика. | **UI:** Невидимо для юзера.<br>**API:** Когда рекламная карточка (получена из `/api/Advertisements` или замешана в `/api/Feed/global`) появляется в Viewport -> `POST /api/Advertisements/{id}/view`. |
