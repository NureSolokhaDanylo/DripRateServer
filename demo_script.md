# Розширений сценарій відеодемонстрації (Скрінкаст)

Цей скрипт побудований як **цілісна історія (User Journey)** одного користувача. Він охоплює всі ключові фічі (гардероб, міні-ігри, стрічки, підписки, рекламу, колекції) та показує їхній взаємозв'язок.

## ЧАСТИНА 1. Клієнтський досвід (Swagger UI)
*Коментар на відео:* "Демонстрацію роботи бекенду я побудую у вигляді реального сценарію використання нашого застосунку."

### 1. Авторизація та "Спроба обману"
* **Дія:** Виконуєте `POST /api/auth/register` (Реєстрація).
* **Коментар:** "Спочатку новий користувач реєструється. Але давайте спробуємо обдурити систему і введемо занадто слабкий пароль або некоректний email."
* **Дані (з помилкою):**
```json
{ "email": "invalid_email", "username": "usr", "password": "123" }
```
* **Дія:** Показуєте 400 Bad Request. 
* **Коментар:** "Як бачимо, `FluentValidation` на рівні Application-шару моментально відхиляє запит. Тепер вводимо валідні дані та логінимося."
* **Дія:** Робите успішний `POST /api/auth/login`, копіюєте JWT-токен у замочок (Authorize).

### 2. Налаштування профілю (Переважні теги)
* **Ендпоінт:** `PUT /api/users/profile`
* **Коментар:** "Користувач налаштовує свій профіль і вказує улюблені стилі одягу (теги). На основі цього алгоритм формуватиме для нього персоналізовану стрічку рекомендацій."
* **Дані:**
```json
{
  "bio": "Люблю вуличний стиль",
  "preferredTagIds": [1, 5, 8]
}
```

### 3. Управління гардеробом (Clothes)
* **Ендпоінт:** `POST /api/clothes`
* **Коментар:** "Перед тим як створити публікацію, користувач має додати речі до свого віртуального гардероба. Бекенд завантажує фото в Azure Blob і зберігає ціну речі."
* **Дані (виконайте 2 рази для створення двох речей):**
```json
{
  "name": "Jordan 4 Retro",
  "category": "Sneakers",
  "estimatedPrice": 350.00,
  "isPublic": true
}
```
* **Дія:** Скопіюйте `Id` створених речей.

### 4. Створення Публікації та отримання Міні-ігор
* **Ендпоінт:** `POST /api/publications`
* **Коментар:** "Тепер користувач створює публікацію і прикріплює до неї свій одяг. Оскільки загальна вартість одягу в луку перевищує певний поріг, система автоматично нараховує йому спроби для міні-ігор."
* **Дані:**
```json
{
  "description": "Мій лук на сьогодні 💧",
  "isUrgentRatingRequested": false,
  "clothIds": ["ВСТАВТЕ_ID_РЕЧІ_1", "ВСТАВТЕ_ID_РЕЧІ_2"],
  "tagIds": [1, 5]
}
```
* **Дія:** Показуєте `200 OK` (публікація створена).

### 5. Міні-ігри та Реєстрація реклами
* **Ендпоінт:** `POST /api/minigames/play`
* **Коментар:** "Завдяки створеній публікації ми отримали спроби для гри. Давайте зіграємо."
* **Дія:** Виконуєте запит, показуєте результат гри (наприклад, виграш віртуальної валюти або бейджа).
* **Ендпоінт:** `POST /api/ads/view`
* **Коментар:** "Коли безкоштовні спроби закінчуються, користувач дивиться нативну рекламу. Цей ендпоінт реєструє перегляд реклами, видає нову спробу і паралельно оновлює Revenue-метрики проєкту."

### 6. Соціальна взаємодія: Пошук та Підписки
* **Ендпоінт:** `GET /api/users/search` (параметр `query=admin`)
* **Коментар:** "Тепер ми знаходимо іншого користувача через пошук і підписуємося на нього."
* **Ендпоінт:** `POST /api/users/{id}/subscribe` (вставте ID знайденого користувача).

### 7. Стрічки новин (Feeds)
* **Коментар:** "У нас є дві різні стрічки новин. Глобальна стрічка базується на алгоритмі рекомендацій (переважних тегах), а стрічка підписок показує контент лише від друзів."
* **Дія 1:** `GET /api/publications/feed` (Глобальна).
* **Дія 2:** `GET /api/publications/feed/subscriptions` (Підписки).
* Показуєте, що ендпоінти працюють і повертають різний контент з курсорною пагінацією.

### 8. Колекції (Collections)
* **Ендпоінт:** `POST /api/collections`
* **Коментар:** "Гортаючи стрічку, користувач знайшов крутий лук і хоче його зберегти. Створюємо кастомну колекцію."
* **Дані:** `{ "name": "Summer Fits", "isPrivate": false }` (Копіюєте ID колекції).
* **Ендпоінт:** `POST /api/collections/{collectionId}/items`
* **Дані:** `{ "publicationId": "ID_ЧУЖОЇ_ПУБЛІКАЦІЇ" }`
* **Коментар:** "Публікацію успішно збережено в колекцію."

## ЧАСТИНА 2. Панель Модератора (Фронтенд React)
*Коментар на відео: "Всі ці активності користувачів (перегляди реклами, скарги, створення контенту) адмініструються через зручний веб-інтерфейс модератора."*

*(Відкриваєте React-застосунок)*

### 9. Дашборд Метрик (Analytics)
* **Дія:** Показуєте головну сторінку з графіками (воронка AARRR).
* **Коментар:** "Зверніть увагу: оскільки ми щойно зареєстрували перегляд реклами через бекенд, наша метрика Revenue тут на графіку автоматично оновилася."

### 10. Управління Нативною Рекламою
* **Дія:** Переходите у вкладку рекламних кампаній.
* **Коментар:** "Тут ми налаштовуємо рекламу, яку щойно дивився користувач. Можна змінити цільові теги або частоту показів у стрічці."

### 11. Репорти (Модерація)
* **Дія:** Відкриваєте таблицю "Репорти".
* **Коментар:** "Якщо хтось створить невідповідну колекцію чи публікацію, ми побачимо це тут."
* **Дія:** Натискаєте "Заблокувати" (Ban).
* **Коментар:** "Система миттєво інвалідує JWT-токен заблокованого користувача, і його сесія завершується. На цьому огляд ключового функціоналу завершено!"
